Quality: About This Data Set

In general, the quality of the data is good because of its correctness. It's complete if we want to use it to answer simple questions, but insufficient to answer more complex questions. Although the raw data is not coherent, but after correct the data type based on domain related knowledge, it become coherent.

Is the data complete?

Whether the data is complete depends on the 'question' we want to answer. If the question is 'Which country won the most goals', then the data is complete. You can group the score team by row counts and rank teams, you will find the answer is Germany (16 goals in all). As shown in chart below.

If the question is 'How star players in a team influence its number of goals', the data is not complete because it lacks a column to specify the whom of those scorers are known as 'star player'.

Is the data coherent?

This data is NOT coherent before cleaning because in the column of 'minute of goal', the field data type of some rows is not correct. It should be a 'number', but appears to be '90+#' or '45+#'. However, according to domain related knowledge (world cup rules), we know the meaning of these data. '45+#' means that the goal was scored in the extra time of the first half match, while '90+#' means the goal was scored in extra time of the second half of the match. Therefore I manually correct these fields by add up the normal time and extra time and reset the range of 'minute of goal' field:
MAX TIME = 45+15+45+15=120 min
MIN TIME > 0 min

Is the data correct?

The data is correct. I verify from three aspects: measure error, fraudulent data and implementation issue.
- Measure error: This data was objective and public, so it has few measure errors.
- Fraudulent data: I use secondary data to verify the correctness of data. From other data source (such as FIFA Offical Website), we know that there 32 teams participated in 64 games and totally scored 145 goal. By comparing the result, the data is proved to be correct.
- Implementation issue: no algorithms were applied to the data and few second time processing traces. So there is few implementation issues.
